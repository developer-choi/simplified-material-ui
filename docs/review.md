# Simplified 문서 리뷰 프로세스

## 목적

단순화된 컴포넌트 코드를 **대화를 통해 깊이 이해**하고, 그 내용을 문서에 반영합니다.

---

## 프로세스

### 1단계: 코드 리뷰 대화

사용자가 컴포넌트에 대해 궁금한 점을 질문합니다.

**예시 질문**:
- "이 부분은 왜 이렇게 구현했어?"
- "이 패턴의 이름이 뭐야?"
- "이거 없으면 어떻게 돼?"
- "원본에서는 어떻게 했어?"

AI는 코드를 분석하고 상세히 설명합니다.

### 2단계: simplified.md 반영

대화에서 나온 유익한 내용을 해당 컴포넌트의 `*-simplified.md`에 반영합니다.

**반영 대상**:
- 새로운 학습 포인트
- 더 명확한 설명
- 추가 시나리오/예시
- 발견된 패턴/플래그

### 3단계: 가이드 문서 반영

반복적으로 유용한 패턴이 발견되면 `docs/format/simplified.md` 양식에도 반영합니다.

**반영 기준**:
- 여러 컴포넌트에서 공통으로 적용 가능한 패턴
- 문서 구조 개선 아이디어
- 새로운 섹션 추가 필요성
